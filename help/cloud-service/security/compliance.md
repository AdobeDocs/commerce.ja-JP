# コンプライアンスと認定制度

[!DNL Adobe Commerce as a Cloud Service]は、信頼性、透明性、およびグローバルなセキュリティとプライバシー標準の厳格なコンプライアンスの基盤の上に構築されています。 Adobeのエンタープライズ レベルのセキュリティ プログラムは、[!DNL Adobe Commerce as a Cloud Service]が規制された業界およびセキュリティに配慮した組織の期待に応えることを保証します。

## 業界の認定制度

Adobeは、[!DNL Adobe Commerce as a Cloud Service]を強化するインフラストラクチャとサービスに関する包括的な認証と証明を保持しています。 次のようなリソースを精査します。

- **SOC 2 Type II**：独立した監査により、セキュリティ、可用性、機密性に関するAdobeの制御が検証されます。

- **ISO/IEC 27001**:AdobeのInformation Security Management System （ISMS）は、この世界的に認められた規格に基づいて認定されています。

- **PCI DSS**: [!DNL Adobe Commerce] インフラストラクチャはPCI DSSに準拠しており、支払いデータの安全な取り扱いを保証します。 顧客には、カスタムコードと拡張機能のPCI コンプライアンスを維持する責任があります。

- **GDPRおよびCCPA**: Adobeには、データプライバシー規制に基づく義務を満たすためのツールとドキュメントが用意されています。

Adobeの認定とコンプライアンスに関するドキュメントの一覧については、[Adobe Trust Center](https://www.adobe.com/trust/compliance/compliance-list.html){target="_blank"}を参照してください。

## 実務担当者を対象とした認定

Adobeは、[!DNL Adobe Commerce]と連携する開発者、アーキテクト、およびビジネス実務担当者に対しても、ロールベースの資格認定を提供しています。 これらの認定は、安全な実装とプラットフォームガバナンスにおける専門知識を証明するものです。 詳しくは、[Adobe認定ポータル &#x200B;](https://experienceleague.adobe.com/en/docs/certification/program/technical-certifications/ac/ac-overview){target="_blank"}を参照してください。
